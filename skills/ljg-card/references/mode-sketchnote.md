# 模具：视觉笔记（-v）

## 核心信条

**一个主题，多视角全景。**

中心是一个核心机制／模型，周围是不同学科的视角卡片，卡片之间用橙色弯曲手绘箭头连接，箭头上写关系。
不是排版好的章节，是一张「知识全景图」——把一个主题在不同学科里如何被看见，全部摊在一张纸上同时显形。

视觉气质不是手绘潦草，是「平静、稳定、信息密度高、有橙色手写体作为生命气息」。

## 形态参考

学科色卡片网格 + 中央核心模型 + 橙色弯曲手绘箭头连接 + 顶部黑色大标题。
卡片自身规整（无衬线黑字、米色底、左侧色条），唯独连接箭头与关系标注用手写体橙色——这是整张图的呼吸。

## 步骤 1：读取模板

Read `assets/sketchnote_template.html`

模板提供（仅"基础设施"，不写死布局）：

- 字体加载：`Noto Sans SC`（主字）/ `Noto Serif SC`（核心卡概念用）/ `Caveat`（手写英文 + 中文回退到楷体）

- `.card .category` `.card .concept` `.card .body` `.card .diagram` 内部结构
- `.title-block`（顶部标题区）
- `.relations`（整页 SVG overlay 容器，画跨卡片箭头用）
- `.relation-label`（橙色手写关系标签）
- `.colophon`（署名 + 来源）
- SVG defs：`arrow-orange`（橙色手绘箭头头）+ `arrow-ink`（卡片内黑色细箭头）

## 步骤 2：理解内容，提炼信息

### 2.1 找一个核心机制／模型（中央卡片）

通读内容，问自己：

> 这段内容在解释**哪一个机制**？如果只能保留一张图，画哪一张？

输出：
- **核心问句**（如"如何表达有价值的内容？"、"语言活动的 3 个维度"）——成为中央卡片标题
- **核心机制图**——一张能浓缩主线索的图（流程／结构／三角／文氏／梯子……）
- **2-4 段说明**——围绕图解释核心论点

**判断标准**：核心卡讲完，其他卡片都是"这个核心放在 X 学科怎么看"。

### 2.2 找 4-8 个不同视角（周围卡片）

为核心机制找学科视角。问自己：

> 这个机制在**哲学／语言学／符号学／认知心理学／逻辑学／法学／社会心理学／语言哲学** 里如何被讨论？
> 内容里有没有显式提到某个领域的具体模型／理论？

每个视角卡片要有：
- **学科分类标签**（小字）：哲学 / 语言学 / 符号学 / 认知心理学 / 法学-论证 / 社会心理学 / 语言哲学 / 逻辑学 / ……
- **概念名**（大粗黑）：拟剧论 / 戴维森三角模型 / 图尔敏模型 / 抽象之梯 / 终极追问 ……
- **图示**：1 个 SVG 简笔图，3-6 个 path，体现该模型核心结构
- **2-4 段说明**：用人话解释这个学科视角看到了什么

**视角数量**：4 个（最少，画面会偏空）—— 8 个（最多，再多塞不下）。默认 5-7 个。

**视角不必都对应学科**——也可以是"步骤"、"层次"、"角色"等内容自然的切分维度。学科色只是 5 种语义色卡，不是"必须填满 5 学科"。

### 2.3 找 3-7 个跨卡片关系（橙色弯曲箭头）

不是所有卡片都两两连。找出能贯穿的"主关系"。问自己：

> 哪两个视角之间有**强关联**？——是"前者解释了后者的某一面"，还是"两者共同支撑了核心机制"？

每条关系包括：
- **起点卡片** → **终点卡片**（哪个指向哪个）
- **关系名**（5-12 字短语，用于标在箭头旁）：如"经验对象的语言化"、"知识生成的逻辑"、"社会互动逻辑"、"记忆加工原理"、"话语组织的规范"

**箭头是叙事的脉络**——读者顺着箭头能看出：从这里看到那里，是因为什么关系。不要硬连。

## 步骤 3：设计画面

### 3.1 色彩使用规范

5 类学科色 + 1 个箭头色，**全图就这 6 种色** + 中性灰黑。

| CSS class | deep（色条 / 标签） | light（卡片底） | 适用学科举例 |
|-----------|---------------------|------------------|---------------|
| `.green`  | `#5B8A6F`           | `#DCE3D5`        | 符号学 / 语言哲学 / 戴维森三角 |
| `.blue`   | `#5979A6`           | `#DCE3EB`        | 语言学 / 抽象之梯 / 行域知域言域 |
| `.red`    | `#B85048`           | `#F0DDD3`        | 哲学 / 认知心理学 / 终极追问 |
| `.gray`   | `#7B7570`           | `#E5E2DC`        | 法学-论证 / 社会心理学 |
| `.amber`  | `#B8924A`           | `#F0E5D2`        | 语言活动维度 / 中性偏暖（亦可作中央备选） |
| `.core`   | 米色 `#F5F1E8`      | 黑色 ink 边     | 中央核心卡（信息密度最高） |

橙色箭头色 `--arrow: #D2693C`：仅用于跨卡片弯曲箭头 + 箭头旁的手写关系标签。

**色彩使用原则**：
- 一类色不重复用太多张——同色卡片之间视觉会粘在一起。同色 ≤ 2 张，最好分开放置
- 中央 `.core` 卡片不染学科色（保持中性米色 `#F5F1E8`），让它"居中而克制"
- 不引入第 7 种色

### 3.2 卡片 HTML 模式

```html
<!-- 普通学科视角卡片 -->
<div class="card green" style="grid-area: a;">
  <div class="category">符号学</div>
  <div class="concept">拟剧论</div>
  <div class="diagram">
    <svg width="280" height="120" viewBox="0 0 280 120">
      <!-- 3-6 个 path，stroke=var(--ink)，stroke-width=2，fill=none 或浅色 -->
      <!-- 体现该理论的核心结构（人物、关系、流程） -->
    </svg>
  </div>
  <div class="body">
    <p>戈夫曼的"拟剧论"试图用戏剧表演理论来说明社交的原理……</p>
    <p>其中角色通过 <strong>四类</strong> 来展开交往：理想化、制造误解、神秘化、补救。</p>
  </div>
</div>

<!-- 中央核心卡片 -->
<div class="card core" style="grid-area: core;">
  <div class="category">核心机制</div>
  <div class="concept">如何表达有价值的内容？</div>
  <div class="body">
    <p><strong>3 个重要变量</strong>：经验值、词汇量、知识量。</p>
    <p>经验值：一切语料来自你切肤感知、特殊的、深刻的经历……</p>
  </div>
  <div class="diagram"><!-- 核心机制图，比普通卡更大 --></div>
</div>
```

### 3.3 SVG 图示设计

每张图给一个 SVG，**3-6 个 path**，体现该理论的核心可视结构。

| 模型类型 | 怎么画 |
|----------|--------|
| 三角／三方关系 | 三个圆 + 连线，每边贴标签 |
| 流程／因果 | 几个矩形 + 箭头连接（用 `marker-end="url(#arrow-ink)"`） |
| 文氏图 | 两三个交叠圆（`opacity:0.5` 半透明色填充） |
| 层级／梯子 | 多条平行线 + 标签 + 一个上下箭头 |
| 五边／多边模型 | 顶点圆 + 连成多边形 + 中心一个圆 |
| 矩阵 | 2x2 网格 + 象限名 |
| 推理链 | 节点 + 横向 → 连接 |

**风格统一**：
- `stroke: var(--ink)` 或 `var(--*-deep)`（同卡片色），`stroke-width: 2`
- `fill: none` 或同卡片浅色 `var(--*-light)` 加 `opacity: 0.6`
- 文字用 `font-family: var(--sans)`，14-18px
- 不要花哨——简笔感，能"立刻看懂结构"

### 3.4 弯曲箭头连接（跨卡片）

整页 `<svg class="relations">` 作为绝对定位 overlay。在里面画 `<path>`，每条 path 后跟一个 `<text>` 标签或 HTML `<div class="relation-label">`。

```html
<svg class="relations" viewBox="0 0 1080 H" preserveAspectRatio="none">
  <!-- 弯曲路径：从卡片 A 边缘连到卡片 B 边缘。三次贝塞尔曲线 -->
  <path d="M 380,460 C 480,440 540,520 620,560"
        stroke="#D2693C" stroke-width="2.5" fill="none"
        marker-end="url(#arrow-orange)" />
</svg>
<!-- 标签是普通绝对定位 div，方便中文断行和字号控制 -->
<div class="relation-label" style="left: 470px; top: 460px; transform: rotate(-6deg);">
  社会互动逻辑
</div>
```

**箭头风格**：
- 一律 **三次贝塞尔曲线**（C 命令）——直线箭头会让画面僵硬
- `stroke-width: 2.5px`，端点用 `arrow-orange` marker
- 路径要从一个卡片的**边缘**指向另一个卡片的**边缘**（不能从卡片内部穿出）
- 同一对卡片之间最多一条
- 全图箭头总数 = 关系数（3-7 条）

**标签风格**：
- 字体 Caveat（中文回退楷体），30px，橙色 `var(--arrow)`
- 旋转角度 `-12deg ~ 12deg`，与箭头大致方向一致
- 紧贴箭头中段上方／下方
- 文字 5-12 字，**不要句号**

### 3.5 整体布局

**画面结构**：

```

│  顶部 .title-block                                       │
│  ┌─ 大标题（90px 黑） ─                                  │
│  └─ 副标题（26px 灰，说明从哪些视角切入）─                 │
│                                                          │
│  ┌──────────────┐  ┌──────────────────────┐  ┌───────┐  │
│  │ .card .green │  │   .card .blue        │  │ .blue │  │
│  │  视角 A      │  │   视角 B（信息密集）  │  │ 视角C │  │
│  └──────────────┘  └──────────────────────┘  └───────┘  │
│         ↘    经验对象的语言化   ↙                          │
│                                                          │
│  ┌─────────────────────────┐  ┌──────────────────────┐  │
│  │   .card .core           │  │  .card .red          │  │
│  │   核心机制问句           │←─│  视角 D              │  │
│  │   核心图 + 详细说明      │  │  （哲学-终极追问）    │  │
│  │   （高度 + 信息最多）    │  └──────────────────────┘  │
│  └─────────────────────────┘                            │
│      ↑ 知识生成的逻辑       ↘ 记忆加工原理                │
│  ┌──────────────┐                                        │
│  │ .card .red   │                                        │
│  │  视角 E（认知）│                                       │
│  └──────────────┘                                        │
│                                                          │
│  ┌──────────┐  ┌──────────┐  ┌────────────────────────┐ │
│  │ .gray    │  │ .gray    │  │ .amber                  │ │
│  │ 视角F法学 │  │ 视角G社心 │  │ 视角H 语言活动维度       │ │
│  └──────────┘  └──────────┘  └────────────────────────┘ │
│           ↑ 话语组织的规范                               │
│                                                          │
│  ─────── colophon ───────                                │
└─────────────────────────────────────────────────────────┘
```

**实现要点**：
- **卡片宽度由内容决定**——不固定列数，不强制对齐。信息量大的卡自然宽，信息少的自然窄
- 推荐用 **flex + flex-wrap** 或 **grid + auto/min-content**——而非 `repeat(N, 1fr)` 这种等分写法
- 通过 `min-width` / `max-width` 给每张卡指定"权重区间"——内容多的 max-width 大，内容少的 max-width 小
- 中央 `.core` 是最大最宽的那张（信息密度最高），通常 `min-width: 600px`
- 卡片之间留 `gap: 16-24px`——既有呼吸又紧凑
- 整体宽度尽量贴 `1080 - 60×2 = 960`，但允许某行不顶满（错落感来自此）
- 整体高度自适应——内容驱动
- 箭头 `<svg class="relations">` 定位在卡片网格之上（`z-index: 50`），坐标根据卡片实际位置标
- **严禁三等分等宽 + 居中 hero**——卡片大小本身就要错落（按内容信息量决定，不按对称美观决定）

## 步骤 4：写 CSS + HTML

把全部 CSS 写入 `{{CUSTOM_CSS}}`，全部 HTML（包括 `.title-block`、卡片网格、`.relations` SVG、`.relation-label`）写入 `{{CONTENT_HTML}}`。

**CSS 模式**（推荐 flex + 内容驱动宽度，禁止 `repeat(N, 1fr)`）：

```css
/* {{CUSTOM_CSS}} */
.title-block { padding: 60px 60px 36px; }
.title-block h1 { font-size: 92px; }

.grid {
  position: relative;
  padding: 0 60px;
  display: flex;
  flex-wrap: wrap;
  gap: 18px;
  align-items: flex-start;
}

/* 卡片基础：宽度由内容驱动，min/max-width 给"权重区间" */
.card {
  flex: 0 1 auto;
  min-width: 240px;
  max-width: 480px;
}

/* 单独给每张卡微调权重——信息多的放宽 max-width，少的收窄 */
.card.core    { min-width: 600px; max-width: 720px; flex-basis: 60%; }
.card.compact { max-width: 260px; }    /* 内容少的卡 */
.card.wide    { max-width: 560px; }    /* 内容稍多的卡 */
```

**布局补充**：
- 不用 `grid-template-areas`——它会强行拉伸卡片到列宽。改用 flex 自然换行
- 卡片排列顺序在 HTML 中决定——希望 .core 出现在视觉中心时，把它放在第 4-5 个位置
- 想让某张卡换行单独占一行？给它 `flex-basis: 100%` 或前面插 `<div style="flex-basis:100%"></div>` 强制断行
- 如果对错落感要求更高：可以分多个 `.row` flex 容器，每行自己换行

**变量替换**：

| 变量 | 内容 |
|------|------|
| `{{CUSTOM_CSS}}` | 全部布局 + 微调（`.grid`、`.title-block` 调整、卡片个性化覆盖） |
| `{{CONTENT_HTML}}` | `.title-block` + `.grid` 容器 + 所有 `.card` + `.relations` SVG + `.relation-label` |
| `{{LOGO}}` | logo 文件路径，使用 `file:///Users/lijigang/.claude/skills/ljg-card/assets/logo.png` |
| `{{SOURCE_LINE}}` | 有来源时填 `<span class="info-source">来源文字</span>`，否则空字符串 |

写入：`/tmp/ljg_cast_sketchnote_{name}.html`

## 与其他模具的边界

- 跟 `-i`（信息图）的区别：信息图是单一主题的数据可视化，本模式是同一主题的**多学科视角并置**
- 跟 `-w`（白板）的区别：白板是推理过程的纵向展开（一条主线），本模式是横向并置（多条视角线）
- 跟旧版 -v（手绘 sketchnote）的区别：旧版追求"像人手画的痕迹"，本模式追求"知识全景的稳定与密度"——只有箭头和标签是手写感

不知道用哪个时，问：「我是要把一件事钻深（→ -w）、还是把一件事铺开多角度（→ -v）、还是讲清一组数据／流程（→ -i）？」
